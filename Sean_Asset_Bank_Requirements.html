Sean’s Website Asset Bank — Client Checklist

This checklist outlines everything we need from you to build the site quickly and smoothly. Please upload assets to a single shared folder and check items off as you go. If something doesn’t exist yet, just mark it “TBD” and we’ll help fill the gap.

How to deliver

  • Preferred: a single shared folder (Google Drive/Dropbox/Notion) with the structure below
  • Grant edit access so we can organize as needed
  • Avoid emailing large files; put them in the folder and notify us

01) Brand basics

  • [ ] Logos (SVG preferred; also PNG @2x). Variants: full, mark-only, dark/bg, light/bg
  • [ ] Color palette with HEX/RGB/CMYK and usage notes
  • [ ] Typography: primary and secondary fonts, files or licenses, sizing guidance
  • [ ] Brand guidelines (if any): voice/tone, spacing, imagery rules

02) Messaging & copy

  • [ ] 1–2 sentence value proposition (what you do, for whom, why it matters)
  • [ ] Elevator pitch (40–60 words) for homepage hero
  • [ ] About/bio (short 75–100 words, long 200–300 words)
  • [ ] Services overview: names, 1–2 line descriptions, inclusions, pricing (if public)
  • [ ] Differentiators/why choose you (3–5 bullets)
  • [ ] Primary calls to action (e.g., Book a call, View work, Download PDF)
  • [ ] Contact copy: email, phone, location, hours, preferred response time
  • [ ] Any boilerplate legal copy you must display

03) Site architecture (pages & navigation)

  • [ ] Sitemap: list of pages (e.g., Home, About, Services, Work/Portfolio, Testimonials, Contact)
  • [ ] Navigation labels and order (top nav + footer links)
  • [ ] Which pages require forms (contact, project brief, newsletter, booking)
  • [ ] 301 redirects (old URLs → new URLs) if this replaces an existing site

04) Portfolio / case studies (if applicable)

For each project/case study, provide:

  • [ ] Title and short summary (1–2 sentences)
  • [ ] Role/services provided
  • [ ] Problem → Approach → Outcome (3 short paragraphs)
  • [ ] 3–8 images (see image specs below) and optional video/embeds
  • [ ] Links (live site, repo, press, social)
  • [ ] Client attribution/permissions (confirm what can be public)

05) Testimonials & press

  • [ ] 3–8 quotes with attribution (name, title, company, headshot if allowed)
  • [ ] Logos of press or client companies (SVG/PNG)
  • [ ] Links to articles or features

06) Team / bio assets (if applicable)

  • [ ] Headshots (min 1600px on short edge, JPG, lightly compressed)
  • [ ] Bios (50–100 words each), roles, social links

07) Visual assets (images & video)

Images

  • [ ] Hero images: 2880×1600px or larger (landscape), JPG, 75–85% quality
  • [ ] Content images: at least 2000px wide, JPG/PNG as appropriate
  • [ ] Transparent artwork/logos as SVG/PNG
  • [ ] Captions/credits if required
  • [ ] Avoid heavy filters/watermarks

Video

  • [ ] MP4 (H.264), 1080p preferred, <= 200MB per clip when possible
  • [ ] Provide a thumbnail/frame recommendation
  • [ ] Hosting preference: upload to YouTube/Vimeo or self-host

08) Downloads / resources (optional)

  • [ ] PDFs, one-pagers, decks, media kit (final, web-optimized)
  • [ ] Any lead magnet or gated content (with form fields you want)

09) SEO & content details

  • [ ] Top 5–10 target keywords/phrases
  • [ ] Per-page meta: title (<=60 chars), description (<=155 chars) if you have drafts
  • [ ] Open Graph image (1200×630px) for social shares
  • [ ] Favicon and app icons (we can generate if you supply a base asset)
  • [ ] Accessibility notes (alt text for key images, contrast requirements)

10) Accounts, integrations, and access

  • [ ] Domain registrar (where your domain is registered) and DNS access or share access
  • [ ] Email provider (for contact form routing)
  • [ ] Google Analytics 4 (GA4) Measurement ID (or we can create)
  • [ ] Google Tag Manager container ID (if used)
  • [ ] Search Console ownership (we can verify post-launch)
  • [ ] Pixels: Facebook, LinkedIn, others (IDs or snippets)
  • [ ] Scheduling tool (Calendly, etc.), newsletter, CRM, payment links, or other embeds

11) Timeline & approvals

  • [ ] Stakeholders for feedback and final approval
  • [ ] Deadlines or launch windows
  • [ ] Blackout dates (vacations, speaking, other commitments)

Image & file specs (quick reference)

  • Photos: JPG, quality 75–85%, sRGB; width ≥ 2000px (heroes ≥ 2880px)
  • Logos/Icons: SVG preferred; if PNG, provide @2x versions on transparent backgrounds
  • Video: MP4 (H.264), 1080p; include a separate thumbnail if possible
  • Documents: PDF (optimized for web); keep under 10MB if possible

What if I’m missing something?

Mark the item as “TBD” and keep moving. We’ll help draft missing copy, suggest images, or prepare interim placeholders so we can keep momentum.
